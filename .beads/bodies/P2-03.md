# P2-03 — Scavenger Training upgrade (1.5-cell radius pull on supplies)

## Summary
Permanent Phase-2-onward upgrade: each tick, supply caches and field rations within 1.5 cells of the patrol head are pulled toward the head at 0.5 × patrol speed (i.e. they move half a cell per tick toward the head). Pulled cells are still grid-aligned per tick — the pull effect manifests as the cache/ration jumping one grid step closer toward the head every ~2 ticks. Pull is paused during Ambush cut animation (§6.4) — for this bead, the Ambush isn't implemented yet, so this is a hook for later (P3-04 will set a `state.midAmbushCut` flag this bead checks).

Pull applies to caches + rations only — NOT to reinforcements, Cover, multipliers, or sprint boosts.

Per PRD §6.4 (Scavenger Training spec).

## Persona
Primary: End User (most visually obvious upgrade — pickups visibly slide toward the patrol).

Skip if the proposed cell is occupied by a patrol segment. |

## Acceptance Criteria
### End User
- [ ] After Phase 2 entry, the player can see supply caches "drift" toward the patrol when within 1.5 cells of the head.
- [ ] Field rations also drift when within 1.5 cells of the head.
- [ ] Pull rate is visibly slower than patrol motion (about half — one step every ~2 ticks).
- [ ] Reinforcement, Cover, multiplier, and sprint boost pickups do NOT drift toward the patrol (later phases — verify this once those exist).

### System
- [ ] `state.scavengerActive` starts false in Phase 1, true from Phase 2 onward, stays true through Phase 5 (permanent for the run).
- [ ] Pull updates respect cell bounds (won't drift off-grid) and patrol occupancy (won't drift into a patrol segment — skip that tick).
- [ ] Pull pauses when `state.midAmbushCut === true` (hook for P3-04 to honor; harmless no-op until P3-04 lands).

## Patterns to Reuse
- Chebyshev distance: `Math.max(Math.abs(dx), Math.abs(dy))`.
- Half-speed via tick counter modulo 2 (no need for fractional pixels — grid-aligned step).
