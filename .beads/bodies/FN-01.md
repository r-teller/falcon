# FN-01 — Bootstrap single-file HTML scaffold + canvas + HUD shell

## Summary
Create the `snake_opus.html` file with inline HTML/CSS/JS scaffolding: a 640×480 px `<canvas>` element inside a thin dark-khaki border frame, an HUD container with placeholder regions (top-left score, top-center phase, top-right reinforcements), and the JS skeleton ready for the game loop and other modules. No external dependencies (no `<script src>`, no `<link rel="stylesheet" href>`, no CDN, no images). Per PRD §1, §3.14, §3.15, §13.1, §13.7.

## Persona
Primary: System (engine scaffold; the player never sees an empty scaffold without the rest of Phase 1 wired in).
Secondary: End User (the canvas frame and HUD shell *will* be visible at run start once the rest of Phase 1 lands).

## Changes Needed
| File | Action | What Changes |
|------|--------|-------------|
| `snake_opus.html` | Create | Single-file HTML document with `<head>` (title, inline `<style>`) and `<body>` (canvas wrapper div + canvas + HUD container + inline `<script>`). Set canvas `width="640"` `height="480"`. Background uses `#2b2d24`; border uses `#4a4632` (per PRD §6.2). HUD typography is monospace/stencil-feel (`font-family: 'Courier New', monospace; color: #e5e0c8`). |

## Acceptance Criteria
### End User
- [ ] Opening the file in Chrome / Edge / Firefox / Safari (modern versions) by double-click renders a dark-olive page with a 640×480 canvas centered, surrounded by a thin dark-khaki border.
- [ ] HUD container is visible with three placeholder regions (top-left, top-center, top-right) styled in parchment monospace text.
- [ ] No external HTTP requests when loading the file (verify in browser devtools Network tab).

### System
- [ ] File is one self-contained `.html` (no sibling JS/CSS/image files).
- [ ] `<script src=...>`, `<link rel="stylesheet" href=...>`, and any CDN URLs are absent.
- [ ] The `<canvas>` element exposes a `getContext('2d')` handle that subsequent modules can target.

## Patterns to Reuse
- File-naming + fallback per PRD §14: `snake_opus.html` (opus is the active model identifier per the system; if uncertain, fall back to `snake_sonnet.html`).
- Aesthetic per PRD §6.2 hex table (background `#2b2d24`, border `#4a4632`, HUD text `#e5e0c8`).

## Testing Strategy
- Manual: open `snake_opus.html` in Chrome, verify visual frame + HUD shell + zero network requests.
- Regression: no prior tests to keep green; this is the genesis file.
