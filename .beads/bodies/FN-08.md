# FN-08 — PATROL LOST screen + R/Space redeploy

## Summary
Implement the run-end screen. When the patrol is lost AND no reinforcements remain (initially: any loss, since reinforcements come in FIN-02/FIN-03), display the PATROL LOST overlay with the canonical copy: headline "PATROL LOST", "Final Score: N", "Phase reached: P", "Press R to redeploy". On Phase 5 also show sector (`Phase 5 — Sector S`). R or Space (per PRD §3.23) redeploys to a fresh Phase 1 run.

Per PRD §3.21 (canonical screen copy); §3.23 (R only from this screen, ignored mid-game); §13.5 (whole-game AC for the screen); §3.24 SECTOR CLEARED variant is OUT of scope here — handled by P5-10.

## Persona
Primary: End User (run-end is a user-visible state).

## Changes Needed
| File | Action | What Changes |
|------|--------|-------------|
| `snake_opus.html` | Edit | Add `state.runState` enum: `'playing' \| 'patrol_lost' \| 'awaiting_first_input'` (default `'awaiting_first_input'`; flips to `'playing'` on first input; flips to `'patrol_lost'` when the loss handler fires). Add HUD overlay `#hud-lost` with the four canonical lines (PATROL LOST / Final Score / Phase reached / Press R to redeploy) styled per PRD §13.5. Extend FN-03 key handler: only when `runState === 'patrol_lost'`, R or Space triggers `redeploy()`. `redeploy()` resets score, patrol (via FN-04 `patrolReset`), supply cache (via FN-05 `spawnSupplyCache`), sets `runState = 'awaiting_first_input'`, hides overlay. Add a `losePatrol()` function: sets `runState = 'patrol_lost'`, shows the overlay, fills in score / phase / (sector if P5). Other modules call `losePatrol()` on lethal events (P1-02 wires wall + self collision). |

## Acceptance Criteria
### End User
- [ ] On patrol loss, the overlay shows EXACTLY: "PATROL LOST" (headline), "Final Score: N" (with current score), "Phase reached: P" (with current phase number; "Phase 5 — Sector S" when in Phase 5), "Press R to redeploy" (bottom).
- [ ] Pressing R from the overlay restarts a fresh Phase 1 run (center, length 3, awaiting first input).
- [ ] Pressing Space has the same effect as R from the overlay.
- [ ] Mid-game R is ignored (no restart) per §3.23.
- [ ] Background is visually dimmed under the overlay so the canvas is still partially visible.

### System
- [ ] Overlay copy strings are EXACT — verify "PATROL LOST" not "GAME OVER" or "YOU DIED"; verify "Press R to redeploy" not "Press R to restart".
- [ ] `redeploy()` fully resets score and patrol; phase resets to 1; reinforcements reset to 0; held bonuses cleared (no bonuses exist yet so no-op for now).

## Patterns to Reuse
- DOM overlay with text content set on display (like FN-07's pause overlay).
- Canonical copy strings stored as constants at the top of the script for easy verification.

## Testing Strategy
- Manual: trigger a loss (drive into a wall — P1-02 wires that), verify each line of canonical copy is exact, press R, verify fresh run starts.
- Manual: during play, press R, verify it is ignored.
- Regression: FN-01..FN-07 still work.
