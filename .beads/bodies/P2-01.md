# P2-01 — Phase 2 camo palette swap on gate (blocky-camo segments, crate-styled supply cache)

## Summary
On entering Phase 2, swap from grayscale night-recon visuals to the full field-ops camo palette: background `#2b2d24`, subtle gridlines `#353727`, patrol body segments alternate (per index) among olive `#556b2f` / tan `#c2b280` / coyote brown `#8b6f47` / black `#1a1a14` so the body reads as blocky camouflage, patrol head darker olive `#3b4a1f` with a directional notch or eye-dot for orientation, supply cache rendered as a tan crate (`#c2b280` fill, `#4a3823` border, optional 1px strap line). The palette stays in effect from Phase 2 through Phase 5 (no jarring swaps later).

Per PRD §1.3 aesthetic; §3.14 visual language; §6.2 palette hex table; §13.7 polish AC.

## Persona
Primary: End User ("dawn breaks" — the most visible single moment in the run).

## Acceptance Criteria
### End User
- [ ] On reaching score 50, the visuals swap from grayscale to camo palette immediately after the phase banner fades (or coincident with it).
- [ ] Patrol body shows at least 3 alternating earth-tone patches across consecutive segments — clearly reads as camouflage, not gradient or single color.
- [ ] Patrol head is visually distinguishable from body (different color AND directional cue — notch or eye-dot pointing in heading direction).
- [ ] Supply cache reads as a small crate or ammo box (tan fill, dark outline, not a fruit / candy / gem).
- [ ] Background is dark olive (`#2b2d24`) with very subtle gridlines (barely visible — terrain texture).
- [ ] No neon or saturated colors appear.

### System
- [ ] `palette()` returns Phase ≥ 2 palette when `state.phase >= 2`.
- [ ] Per-segment body tone is deterministic on segment index (so the camo pattern is stable; not flickering per frame).
