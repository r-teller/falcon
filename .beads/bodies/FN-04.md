# FN-04 — Patrol data model (segments, head, growth, length math)

## Summary
Implement the snake/patrol data model: an array of `{x, y}` cell coordinates where index 0 is the head and the tail is at the end. Provide `step(direction)` that advances the head and either truncates the tail (no growth) or keeps it (growth). Provide `grow(n)` to schedule N growth ticks. Provide `reset(cx, cy, length, heading)` for fresh spawn (called by FN-08 / phase respawn). Wire FN-04 into FN-02's `tick()` callback to drive movement, gated on `state.firstInputReceived` per PRD §3.39 for Phase 1.

Per PRD §3.20 (spawn: center, length 3, heading right); §4.4 (growth +1 per supply); §3.39 (Phase 1 waits for first input; respawn auto-resumes).

## Persona
Primary: System (data model).
Secondary: End User (patrol movement is the most visible mechanic).

## Changes Needed
| File | Action | What Changes |

| `snake_opus.html` | Edit | Add `state.patrol` object: `segments: [{x,y}, …]` (head first), `pendingGrowth: int`. Add `patrolStep()` called from `tick()`: read `state.pendingDirection ?? state.currentDirection`, commit it to `state.currentDirection`, compute new head, prepend to `segments`, then if `pendingGrowth > 0` decrement it (no truncate), else pop the tail. In Phase 1, `tick()` skips `patrolStep()` until `state.firstInputReceived` is true. Add a simple draw routine for segments (placeholder gray; P1-01 will replace). |

## Acceptance Criteria
### System
- [ ] `patrolReset()` produces 3 segments at the center cells (e.g. cells (16,12), (15,12), (14,12)) with head at (16,12) and `currentDirection = 'right'`.
- [ ] `patrolStep()` with `pendingDirection = 'right'` advances head from (16,12) to (17,12), drops tail (14,12) — patrol stays length 3.
- [ ] After `grow(1)` and one `patrolStep()`, length is 4 (tail retained).
- [ ] `patrolStep()` is not invoked while `firstInputReceived === false` (Phase 1 only — wait state).

### End User
- [ ] On page load, a 3-segment placeholder patrol is visible at the center.
- [ ] After pressing an arrow key, the patrol begins moving.

## Patterns to Reuse
- Standard linked-list-style snake body, but using a plain array. Head-first ordering simplifies collision checks later (just iterate `segments.slice(1)`).

## Testing Strategy
- Manual: open page, see 3 segments at center, press Right Arrow, watch patrol move right. Verify length stays 3 (no growth yet — FN-05 wires growth on pickup).
- Console probe: `state.patrol.segments.length` before/after a Right Arrow + few ticks.
- Regression: FN-02 loop, FN-03 input keep working.
