# P1-04 — Phase 1→2 gate detection + banner

## Summary
When score reaches 50, transition from Phase 1 to Phase 2: set `state.phase = 2`, set `state.phaseName = 'PHASE 2: DAWN PATROL'`, re-baseline tick rate (the formula for Phase 2 is again `6 + floor((score - 50) / 25)` per the standard in-phase ramp), and trigger a 2-second phase-transition banner via FIN-01 (or a local shim if FIN-01 isn't done yet — see Patterns). Score, patrol position, length, direction, and reinforcements all persist per PRD §3.19. The banner appears WITHOUT pausing the game (patrol keeps moving) per §3.37.

This bead lays the gate-detection pattern that P2-05, P3-07, and P4-07 will copy.

Per PRD §3.1, §3.19, §3.37, §5.4, §12.

## Persona
Primary: End User (phase transitions are the visible reward of progress).

Banner: if FIN-01 is done, call `showPhaseBanner(name)`; otherwise local 2-second `setTimeout`-driven DOM overlay (acceptable interim). |

## Acceptance Criteria
### End User
- [ ] On reaching score 50 in Phase 1, a 2-second banner reading "PHASE 2: DAWN PATROL" appears centered over the canvas, then fades.
- [ ] During the banner, the patrol keeps moving (per §3.37 — the game does not pause).
- [ ] After the transition, the HUD phase text reads "PHASE 2: DAWN PATROL" (or equivalent).
- [ ] Score, patrol length, direction, and position are all preserved across the transition (verify: collect a supply at score 49, then the next one immediately triggers transition; patrol does NOT reset).

### System
- [ ] `transitionToPhase()` is generic (takes phase number + display name) so later gates (P2-05, P3-07, P4-07) reuse it.
- [ ] `checkPhaseGates()` is idempotent — once `state.phase === 2`, the Phase-1 gate branch no longer fires.
- [ ] `recomputeTickRate()` is called after phase change so the speed ramp re-baselines from the new phase floor.

## Patterns to Reuse
- Reuse the `palette()` / `recomputeTickRate()` phase-branch pattern.
- Banner display: if FIN-01 isn't landed yet, write a minimal local banner — a centered overlay div with the text, displayed for 2000 ms with a CSS fade (`opacity: 1 → 0`). FIN-01 will generalize.
