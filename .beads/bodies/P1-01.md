# P1-01 — Wire grayscale night-recon palette + Phase 1 visuals

## Summary
Apply the Phase 1 grayscale night-recon styling to all visible elements during Phase 1: near-black background (`#0e0e0e` or dark gray on the canvas, distinct from Phase 2's `#2b2d24` dark olive), light-gray patrol on dark gray gridlines (1px at 10% opacity), supply cache slightly lighter than background. NO colored elements. The grayscale only applies in Phase 1; the Phase 2 gate (P2-01) restores the camo palette. Per PRD §5.2, §13.7.

## Persona
Primary: End User (palette is the immediate visual cue that Phase 1 is the night-recon training mood).

## Changes Needed
| File | Action | What Changes |
|------|--------|-------------|
| `snake_opus.html` | Edit | Introduce a `palette()` helper that returns colors based on `state.phase`. Phase 1 returns: `bg='#0e0e0e'`, `gridline='rgba(180,180,180,0.10)'`, `patrolHead='#cccccc'`, `patrolBody='#999999'` (single gray tone — no alternating camo in Phase 1), `supplyCache='#3a3a3a'`. The canvas clear + draws in FN-01..FN-05 must consult `palette()` rather than hardcode colors. Draw subtle gridlines on canvas during render. Patrol body uses a single gray tone for all segments in Phase 1 (alternating earth-tone patches come in P2-01). |

## Acceptance Criteria
### End User
- [ ] Phase 1 background is near-black (visually distinct from the dark-olive Phase 2 background).
- [ ] Subtle gray gridlines are visible on the canvas at ~10% opacity (not stark, not absent).
- [ ] Patrol renders in a single light-gray tone, head slightly lighter than body (still distinguishable).
- [ ] Supply cache is a flat light-gray-on-dark cell that reads against the terrain.
- [ ] No colored elements appear in Phase 1 (no greens, browns, golds — strictly grayscale).
- [ ] The Phase 1 "night-recon / training-exercise" mood is unambiguous on first frame.

### System
- [ ] `palette()` is a single source of truth — all draw functions consult it rather than hardcoding colors.
- [ ] Switching `state.phase` from 1 to 2 (manually via console for verification) causes the next render to swap to Phase 2 colors (P2-01 will own the actual swap; verifying `palette()` plumbing here).

## Patterns to Reuse
- Switch over `state.phase` inside `palette()`: a `case` per phase. Future phases extend the same helper.
- Gridline drawing: a single loop of vertical + horizontal lines, drawn once per frame on the canvas at low opacity.

## Testing Strategy
- Manual: open the page, verify the Phase 1 grayscale mood. Temporarily set `state.phase = 2` in the console, refresh draws, confirm a palette switch is plumbed (Phase 2 visuals are placeholder until P2-01).
- Manual: ensure no neon / saturated colors are visible in Phase 1.
- Regression: FN-01..FN-05 still work; collisions still trigger PATROL LOST etc.
