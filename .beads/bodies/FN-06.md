# FN-06 — HUD renderer (score, reinforcements, phase, held-bonus icons)

## Summary
Implement the HUD render layer: draw the score at top-left, the reinforcements row (squad-marker chevrons, max 5) at top-right, and the current phase name in the top-center, per PRD §13.4. Reserve render slots for held-bonus icons (Cover, sprint boost, active multiplier) — these go unused until later phases (P3, P4, P2 respectively) but the structure is in place. The HUD can either be DOM (overlaid divs) or canvas-drawn; DOM is simpler and keeps the canvas focused on the playfield — recommended.

Per PRD §3.21 (PATROL LOST is a separate screen — FN-08 owns it; FN-06 is gameplay HUD only); §3.28 (squad-marker chevrons; heart icon acceptable if chevron unreadable); §13.4 (HUD spec).

## Persona
Primary: End User (HUD is constantly visible during play).

## Changes Needed
| File | Action | What Changes |
|------|--------|-------------|
| `snake_opus.html` | Edit | In `<body>`, add HUD container divs: `#hud-score` (top-left), `#hud-phase` (top-center), `#hud-reinforcements` (top-right), `#hud-bonuses` (below phase). CSS positions them absolutely over the canvas with parchment monospace styling. Add `renderHud()` called from the render loop: writes `state.score`, `state.phaseName` (default "PHASE 1: NIGHT RECON"), and renders `state.reinforcements` (init 0) as N chevron glyphs (Unicode `▼` or `⌃` works; or a small SVG/CSS shape). `#hud-bonuses` is empty for now — P2/P3/P4 will populate. |

## Acceptance Criteria
### End User
- [ ] Score (starting "0") is visible top-left and updates when supply caches are collected.
- [ ] Current phase name is visible top-center.
- [ ] Reinforcements area is visible top-right (initially empty since reserve starts at 0; FIN-02 adds chevrons on pickup).
- [ ] HUD text uses parchment/monospace styling consistent with PRD §6.2 (`#e5e0c8`).

### System
- [ ] `renderHud()` is called every render frame (so score updates immediately on cache pickup).
- [ ] HUD never occludes the playfield (`pointer-events: none` on overlay divs).

## Patterns to Reuse
- DOM overlay over canvas: position the canvas in a wrapper `<div>` with `position: relative`, HUD divs `position: absolute` over it with `pointer-events: none`.
- Reinforcement chevrons: a simple flexbox row with N child elements; rebuild on every render (cheap for max 5 items).

## Testing Strategy
- Manual: collect supply caches and verify the score in the HUD updates. Verify phase name is readable. Verify the reinforcements slot is positioned correctly even when empty.
- Regression: canvas playfield (FN-01..FN-05) still renders cleanly.
