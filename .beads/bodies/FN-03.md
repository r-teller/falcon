# FN-03 — Input handler (arrows + WASD + direction reversal protection)

## Summary
Listen for `keydown` events on `window` and translate Arrow keys + WASD into a `pendingDirection` value (up/down/left/right). On each tick, FN-04 will read `pendingDirection` and apply it. Enforce two protections per PRD §3.18: (1) only one direction change applies per tick (no rapid double-input), (2) inputs that are the direct opposite of the current heading are rejected (not queued).

Per PRD §4.3 (controls); §3.18 (reversal protection); §3.39 (Phase 1 waits for first input — handled by FN-04 reading `firstInputReceived` flag this module sets).

## Persona
Primary: End User (player input is the primary interaction surface).

## Acceptance Criteria
### End User
- [ ] ArrowUp/ArrowDown/ArrowLeft/ArrowRight set `pendingDirection` to up/down/left/right.
- [ ] W/A/S/D (case-insensitive) set the same directions.
- [ ] Pressing the direct opposite of `currentDirection` (e.g. Right while heading Left) leaves `pendingDirection` unchanged.
- [ ] Arrow keys do not scroll the page.
- [ ] First valid direction input sets `firstInputReceived = true`.

### System
- [ ] Multiple direction inputs between ticks resolve to the latest VALID input (not the latest input overall) — so a Right→Up→Down sequence while heading Left lands on Up (Right was rejected as opposite; Down is allowed).
- [ ] P, Esc, R, Space, Shift handlers exist (may stub `console.log` for now); FN-07 / FN-08 / P4-06 will replace.

## Patterns to Reuse
